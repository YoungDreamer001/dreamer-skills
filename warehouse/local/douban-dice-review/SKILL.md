# 骰子豆瓣影评

根据用户给出的电影名称，检索电影信息，用 6 个骰子逐轮投掷三轮；每投完一轮，立刻向用户呈现该轮结果，并马上写出对应影评。三轮结束后，生成 3 份中文豆瓣风格影评，并把全过程产物写入同一个输出目录。

## 输出合同

每次成功执行都创建一个影评包目录。目录名使用电影名清理后的短 slug，例如 `哪吒_骰子影评/`。所有文件必须写入该目录，不要散落在当前工作目录。

固定产物：

```text
[电影名]_骰子影评/
├── movie_info.md
├── draft.md
└── 影评.md
```

最终给用户呈现 `影评.md` 中的 3 份影评，并报告影评包路径。除非用户明确只要对话输出，否则仍然必须写入文件。

## 工作流

### 1. 检索电影信息

搜索用户输入的电影名称，优先获取：

- 片名、年份、导演、主要演员；
- 类型、国家或地区、片长；
- 可靠故事梗概；
- 与影评写作相关的主题、人物关系、视觉或叙事特征。

把检索结果写入 `movie_info.md`。要求：

- 用中文整理；
- 标注信息来源或搜索依据；
- 区分确定事实与不确定信息；
- 不要把无来源的影评观点写成电影事实。

如果无法联网或搜索失败，直接说明限制，并基于用户已给信息写入 `movie_info.md`。不要伪造真实资料。

### 2. 逐轮掷骰并写草稿

每一轮单独运行一次脚本。不要一次性投完三轮再统一呈现。

```bash
python3 [skill-dir]/scripts/roll_dice.py --round 1 --output [影评包目录]/draft.md
python3 [skill-dir]/scripts/roll_dice.py --round 2 --output [影评包目录]/draft.md
python3 [skill-dir]/scripts/roll_dice.py --round 3 --output [影评包目录]/draft.md
```

如需可复现结果，可追加 `--seed 任意字符串`。脚本必须在终端逐条输出本轮投掷结果，例如 `第一轮第一个骰子结果：齐泽克`，最后输出本轮汇总，例如 `第一轮骰子结果：1齐泽克 2身份 3家庭 4他者 5后现代性 6身份认同危机`。脚本同时把本轮投掷过程追加写入 `draft.md`，让用户能看到过程。

六个骰子固定如下：

1. 理论家：1 弗洛伊德，2 齐泽克，3 拉康，4 弗洛伊德，5 齐泽克，6 拉康
2. 电影讨论的是啥：1 欲望，2 创伤，3 记忆，4 身份，5 权力，6 异化
3. 分析对象：1 身体，2 家庭，3 空间，4 时间，5 消费，6 语言
4. 高级名词：1 他者，2 凝视，3 缺席，4 能指，5 规训，6 主体
5. 格局打开：1 现代性，2 后现代性，3 消费社会，4 资本主义，5 数字时代，6 景观社会
6. 给出结论：1 主体性的瓦解，2 身份认同危机，3 归属感消失，4 现实边界模糊，5 日常经验陌异化，6 意义系统崩塌

每投完一轮，就在 `draft.md` 继续追加该轮写作草稿，并马上把对应影评写入 `影评.md`。不要等三轮全部投完后才开始写影评。每轮草稿至少写 3-6 句，必须把该轮 6 个关键词全部纳入论证链。

### 3. 写成影评文档

根据 `movie_info.md` 和 `draft.md` 写 `影评.md`。每轮骰子对应一份影评。

`影评.md` 固定结构：

```markdown
# 《电影名》骰子影评

## 影评一

**骰子结果**：[理论家] / [议题] / [分析对象] / [高级名词] / [时代框架] / [结论]

[影评正文]

## 影评二

**骰子结果**：...

[影评正文]

## 影评三

**骰子结果**：...

[影评正文]
```

每份影评控制在 250-600 字。优先写得像豆瓣短评和长评之间的中文影评：有判断、有文本细节、有理论腔，但不要写成论文摘要。

## 写作规则

- 用中文写。
- 不要解释“我在使用骰子”之外的流程废话。
- 每份影评必须引用至少一个电影具体元素：人物、场景、情节、叙事结构、视觉风格、声音、台词或类型套路。
- 理论词汇要服务于电影，不要只堆名词。
- 可以讽刺、冷峻、玄学、不可证伪，但要保持句子顺。
- 不要把错误剧情当事实；资料不确定时，用“如果按公开梗概理解”这类限定。
- 三份影评的角度要明显不同，不能只是替换关键词。
- 结尾要落到第 6 个骰子的结论词。

## 失败处理

如果电影名有多个同名条目，选择最有可能的电影并在 `movie_info.md` 说明；歧义会严重影响结果时，先问用户一个问题。

如果脚本无法运行，手动投掷三轮并在 `draft.md` 明确记录骰子点数与关键词，但仍要完成 `影评.md`。
